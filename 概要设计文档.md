## 1. 引言

### 1.1 编写目的

本概要设计文档旨在对"跨模态 RAG+SQL 双分支智能问答系统"进行总体设计说明，为后续详细设计和编码实现提供指导。文档面向开发人员、测试人员和项目管理人员。

### 1.2 项目背景

本系统是一个面向海事领域的智能知识管理与问答平台，结合了向量检索（RAG）和结构化查询（SQL）两种检索范式，支持多种数据格式的知识入库、语义检索和智能问答。系统特别针对港口、航线、规章制度等海事专业数据进行优化。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| RAG | Retrieval-Augmented Generation，检索增强生成 |
| Embedding | 向量化/嵌入，将文本转换为高维向量表示 |
| pgvector | PostgreSQL的向量扩展插件 |
| SSE | Server-Sent Events，服务器推送事件 |
| LLM | Large Language Model，大语言模型 |

## 2. 系统概述

### 2.1 系统目标

1. **多模态知识管理**：支持文本、图片、Excel、Word、JSON等多种格式
2. **双分支检索**：向量语义检索 + SQL结构化查询
3. **海事专业支持**：港口、航线、航迹数据的专门处理
4. **智能问答**：基于检索结果的流式问答生成
5. **海图可视化**：港口和航线的地图展示

### 2.2 支持的数据格式

| 类型 | 格式 | 说明 |
|------|------|------|
| 文本 | .txt | 纯文本文件 |
| 表格 | .xlsx, .xls, .csv | Excel和CSV表格数据 |
| 文档 | .docx | Word文档 |
| 标记 | .md | Markdown文档 |
| 结构化 | .json | JSON数据文件 |
| 图片 | .png, .jpg, .jpeg, .gif, .webp | 图片文件（支持OCR） |
| 航迹 | 自定义格式 | 港口、航线空间数据 |

## 4. 功能模块设计

### 4.2 模块详细设计

#### 4.2.1 文件上传与解析模块 (M01)

**功能描述：**
- 支持多格式文件上传
- 自动识别文件类型并调用对应解析器
- 解析结果存入知识库

**处理流程：**
```
文件上传 → 类型识别 → 解析器选择 → 内容提取 → 数据入库
```

**支持格式与解析方式：**
| 格式 | 解析方式 |
|------|----------|
| .txt | 直接读取文本 |
| .pdf | PDF文本提取 |
| .docx | Word文档解析 |
| .xlsx/.xls | Excel表格解析 |
| .csv | CSV表格解析 |
| .json | JSON结构解析 |
| .md | Markdown解析 |
| 图片 | AI视觉理解 |

#### 4.2.2 向量化处理模块 (M02)

**功能描述：**
- 对知识条目进行向量化处理
- 支持2048维向量存储
- 支持批量处理和增量处理

**处理流程：**
```
获取待向量化条目 → 文本预处理 → Embedding API调用 → 向量存储
```

**向量化策略：**
- 文本条目：直接向量化
- 图片条目：图像描述后向量化
- 港口数据：组合名称+国家+位置信息
- 航线数据：组合起点+终点+航线信息

#### 4.2.3 知识检索模块 (M03)

**功能描述：**
- 双分支检索架构
- 向量语义检索（RAG分支）
- SQL结构化查询（SQL分支）
- 结果融合与排序

**检索流程：**
```
用户查询 → 意图识别 → 分支选择
    │                    │
    ▼                    ▼
向量检索              SQL查询
(语义相似度)          (精确匹配)
    │                    │
    └────────┬───────────┘
             ▼
         结果融合 → 排序 → 返回
```

**检索源：**
| 数据源 | 检索方式 | 说明 |
|--------|----------|------|
| knowledge_items | 向量检索 | 通用知识条目 |
| port_data | 向量检索 | 港口数据 |
| route_data | 向量检索 | 航线数据 |
| 规章制度表 | 向量检索 | 制度文档 |

#### 4.2.4 智能问答模块 (M04)

**功能描述：**
- 基于检索结果的问答生成
- 支持流式输出（SSE）
- 支持上下文对话

**问答流程：**
```
用户问题 → 知识检索 → 上下文构建 → LLM推理 → 流式输出
```

**LLM调用方式：**
- 使用SSE协议实现流式传输
- 后端通过ReadableStream逐步输出
- 前端通过Reader逐步读取渲染

#### 4.2.5 数据维护模块 (M05)

**功能描述：**
- 港口数据管理（CRUD、导入、向量化）
- 航线数据管理（CRUD、导入、向量化）
- 规章制度管理（上传、分类、向量化）
- 海图统计（区域分布直方图）

**功能列表：**
| 功能 | 港口 | 航线 | 规章制度 |
|------|------|------|----------|
| 列表查询 | ✓ | ✓ | ✓ |
| 模糊搜索 | ✓ | ✓ | ✓ |

| 新增/编辑 | ✓ | ✓ | - |

#### 4.2.6 海图显示模块 (M06)

**功能描述：**
- 港口位置地图标注
- 航线轨迹绘制

```
POST /api/upload

Request:
  - file: 文件

{
  "success": true,
  "items": [...],
  "fileId": "xxx"
```

```
POST /api/search
Content-Type: application/json

Request:
{
  "query": "搜索关键词",

  "threshold": 0.3
}

{
  "success": true,
  "query": "搜索关键词",
  "results": [
    {
      "id": "xxx",
      "modality": "port",
      "title": "金兰湾",

      "similarity": 0.65
    }
  ]

```
